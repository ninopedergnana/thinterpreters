GOTO

Syntax


GOTO Programs consists of a sequence of instructions I, each introduced by a marker Mi:.
In order to improve the readability of GOTO programs, you usually write the instructions in enumerated order of the markers, with each instruction on a separate line of code.

The following options are permitted as possible instructions.

Addition

The variable xi is assigned the value of xj + c

Subtraction

The variable xi is assigned the value of xj - c or the value 0 if the assignment tries to assign a negative value

Unconditional jump

The code continues at Mi

Conditional jump

If the condition is met, the code continues with Mi, otherwise it will continue with the next line M(i + 1)

Halt

Halt stops the program and returns x0

Additional Information

  • The variables xi and xj can be the same. You can use x1 = x1 + 1 to increment x1 by 1 for example. The constant c can only be an Integer value.

  • If there is no jump label to a new marker, the program will continue with the next line.

  • If the last line is reached and there is no jump label, the program will halt and return x0.

  • PARSE will only parse the source code, not run the program. RUN will run the program with the given inputs.

  • The result will always be returned in x0 with a default value of 0.

Error handling

If your code contains a syntax error, the editor will highlight the faulty line for you. Try to fix it and run the code again. If you're stuck, consider consulting the examples for help.
If you implemented an infinite loop, the process will terminate after 5 seconds and notify you of your mistake.
If your code deviates from the specified behaviour and you don't get the expected result without having received any syntax error messages, you probably implemented a logical mistake in your source code. The interpreter can't catch these types of mistakes. You're on your own :)

LOOP

Syntax

Loop programs constist of the following syntactical basic elements.

  • Variables: x0, x1, x2, ...

  • Constants: 0, 1, 2, 3, ...

  • Separator: ;

  • Assignment: =

  • Operation Signs: +, - ;

  • Key words: Loop, Do, End

Assignments: x = a + b and x = a - b, where x is a variable and a and b stand for variables or constants.
Sequences: When P and Q are LOOP program, P;Q is also a LOOP program.
Loops: If P is a LOOP program, then for each variable x: "LOOP x DO P END" is also a loop progam.

Loop programs begin with the keyword "Loop" and end with the keyword "End".
The literal between the "Loop" and the "Do" keyword indicates the number of iterations of the subroutine. If the literal has a value smaller than or equal to zero, the subroutine won't be executed.

WHILE

Definition

If the loop language is extended by the additional syntactic module WHILE xi > 0 DO ... END for all variables xi, then you get the set of all WHILE programs.
Every LOOP program is also a WHILE program.
WHILE programs don't always terminate. Under certain circumstances, they can run forever.
The semantics of WHILE programs is analogous to the ones of LOOP programs.

Syntax

Literals

Literals are either variables or constants.

Variables

Variables start with the letter x and are followed by a natural number, i.e. (x0, x1, x2, ...). Variables starting with any different letter will not be recognized as such by the parser.
Depending on the number of parameters for your program, the first n variables after x0 will be assigned with the corresponding values. Let's say your program needs 3 parameters. The three values typed into the parameter field will be assigned to the variables x1, x2, x3 accordingly. x0 has the function of the return value (the result of your program). The remaining variables (x4, x5, x6, ...) can be used freely.
After parsing, variables will have the prettyfied form "X i" for i in 0, 1, 2, ... .
If not defined, variables are initialised with the value 0.

Constants

Since the parser is based on the set of natural numbers (0, 1, 2, ...), constants are limited to these numbers.
After parsing, constants will have the prettyfied form "C i" for i in 0, 1, 2, ... .

Terms

Terms are simple arithmetic expressions of literals.
After parsing, terms will be written in prefix notation, starting with the operation and followed by the operands.
Please note that arithmetic expressions are binary operations and therefore always need two operands.
Plus denotes the addition of two natural numbers, i.e. x2 + 1 parses to "Plus (X 2) (C 1)".
Minus denotes the subtraction of two natural numbers, i.e. x3 - x1 parses to "Minus (X 3) (X 1)".
Multiplication and division are not recognized by the parser. Nevertheless, you can write a program for these operations see "Examples"

Programs

A program is one of the following constructs:

  • Empty program
  • Assignment
  • Loop program
  • While program
  • Sequence of programs

Assignments

An assignment has the form variable = term.
Please note that terms are based on binary arithmetic expressions. As a consequence, it is not possible to assign a variable to a literal directly. Instead, transform the literal into a Term.
